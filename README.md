# ultraCRM 💻

Полноценная CRM для сервисного центра по ремонту техники (телефоны, ноутбуки, ТВ, бытовая техника — 400+ типов устройств через настраиваемые категории). Архитектура и UX вдохновлены Workpan: приёмка заказа **меньше чем за 60 секунд с телефона**, цветные статусы, печать документов из браузера.

**Стоимость инфраструктуры: 0 ₽/мес** (Supabase Free + бесплатный хостинг статики). Команда: 1–10 сотрудников.

## Возможности

- **Заказы**: приёмка одним экраном, машина статусов (10 статусов / 22 перехода, настраивается в БД), работы и запчасти, предоплата и оплата, просрочки, история изменений (append-only).
- **Клиенты**: дедупликация по телефону, история обращений, задачи «позвонить» на дашборде.
- **Справочник**: категории/бренды/модели, быстрое добавление при приёмке, импорт из Excel/CSV, динамические доп-поля по категориям (IMEI, диагональ…) без миграций.
- **Роли**: админ / менеджер / мастер — тройная защита (RLS на строки, триггеры на колонки, скрытие в UI). Мастер видит только свои заказы и не видит финансов.
- **Фото**: съёмка с телефона, сжатие на клиенте до ~250 КБ, приватное хранилище, галерея.
- **Документы**: квитанция с QR, акт работ, акт выдачи, гарантийный талон — печать/PDF из браузера, воспроизводимость из снимков.
- **Клиентский сервис**: публичная страница статуса по QR (без входа, белый список полей), уведомления Telegram / Email / задача «позвонить» с идемпотентной доставкой.
- **PWA**: установка на телефон, офлайн-чтение, офлайн-очередь смены статусов.
- **Тёмная тема** — единственная и основная.

## Стек

| Слой | Технологии |
|---|---|
| База | PostgreSQL (Supabase): 10 миграций, ~2000 строк SQL — RLS, триггеры, RPC, state machine |
| Бэкенд | Supabase (PostgREST, GoTrue, Storage) + 4 Edge Functions (Deno): уведомления, публичный статус, Telegram-вебхук, управление сотрудниками |
| Фронтенд | React 19 + TypeScript strict + Vite + Tailwind 4 + TanStack Query (35 файлов, ~4300 строк) |
| Тесты | 28 SQL-проверок (RLS + бизнес-логика) на живом PostgreSQL + 5 Deno-тестов + демо-данные как интеграционный тест |
| CI/CD | GitHub Actions: сборка, все тесты, авто-деплой бэкенда в Supabase |

## Быстрый старт

```bash
# Локально (нужны Docker + Supabase CLI):
./scripts/dev.sh        # стек + миграции + демо-данные + .env.local
npm install && npm run dev
# Вход: admin@demo.local / manager@demo.local / master@demo.local, пароль demo1234
```

**Развёртывание в продакшен**: [docs/DEPLOY.md](docs/DEPLOY.md) — с нуля до работающей системы за 40–60 минут.

## Команды

```bash
npm run dev              # разработка
npm run build            # tsc strict + vite build (+PWA)
npm run test:db          # 28 проверок БД + интеграция демо-данных
npm run test:functions   # Deno-тесты шаблонов уведомлений
npm run icons            # перегенерация PWA-иконок из SVG
```

Проект разработан поэтапно, каждый этап задокументирован с решениями и рисками: [docs/stages/](docs/stages/) — от анализа требований (этап 1) до финального аудита (этап 18). Деплой: [docs/DEPLOY.md](docs/DEPLOY.md).

## Структура

```
supabase/
  migrations/   # 10 миграций: схема, триггеры, RPC, индексы, RLS, роли, Storage
  functions/    # Edge Functions: notify-dispatch, public-status, telegram-webhook, admin-users
  tests/        # RLS-матрица (15) + бизнес-логика (13) на живом PostgreSQL
  seed.sql      # статусы, переходы, правила уведомлений, шаблоны полей
src/
  app/          # роутер, авторизация, layout, офлайн-баннер
  features/     # заказы, клиенты, справочник, настройки, дашборд, публичная страница
  shared/       # API-клиенты, UI-кит, утилиты
scripts/        # dev.sh, test-db.sh, demo-data.sql, generate-icons.mjs
docs/           # DEPLOY.md + 18 этапов разработки
```
