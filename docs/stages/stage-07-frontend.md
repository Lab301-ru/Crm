## 2. Тёмная тема

Токены в `@theme` (Tailwind 4): фон `#0b0f17`, поверхности `#121826`/`#1a2233`, границы `#243047`, текст `#e6eaf2`, приглушённый `#8b97ab`, primary `#3b82f6`. `color-scheme: dark` — нативные контролы (date picker, select) тоже тёмные. Цвета статусов приходят **из БД** (`statuses.color`) и рендерятся бейджами с полупрозрачной подложкой — контрастно на тёмном фоне, переживёт любые новые статусы.

## 3. Workpan-паттерны в UI (UX-N1)

| Привычка | Реализация |
|---|---|
| Цветные статусы везде | StatusBadge в таблице, карточке, поиске, публичной странице |
| Строка статусов-фильтров над таблицей | чипы статусов на OrdersPage, активный — в цвет статуса |
| Поиск «по куску» номера/телефона/IMEI | глобальный поиск с подсказкой «найдено по: …» |
| Приёмка одним экраном | NewOrderPage: телефон → клиент подтянулся → категория → бренд/модель из автодополнения → неисправность → «Принять в ремонт» |
| Модели нет — добавил на лету | кнопка «+ Добавить “бренд модель” в справочник» прямо в форме |
| Работы и материалы строками в наряде | ItemsCard с типом строки, итоги пересчитываются с сервера |
| Кнопки переходов = state machine | в карточке рендерятся только разрешённые переходы (из `status_transitions`), в цветах целевых статусов |
| Просрочка | красный бейдж «Просрочен» в списках и карточке |

## 4. Ключевые решения

- **Состояние = TanStack Query, без Redux:** все данные серверные; кэш с инвалидацией по мутациям. Локальный стейт — только формы.
- **API-слой тонкий и типизированный** (`shared/api/*`): каждый вызов возвращает доменные типы, ошибки БД пробрасываются как есть — русские сообщения из PostgreSQL показываются пользователю (`ErrorText`).
- **Мобайл-ферст:** нижняя навигация с крупной центральной кнопкой «+» (новый заказ — главное действие), таблица заказов на телефоне превращается в карточки, `tel:` ссылки для звонка клиенту, safe-area для iPhone.
- **Тяжёлые парсеры импорта (xlsx 143 КБ gzip) — динамический import** в момент выбора файла: менеджеры и мастера их никогда не загружают.
- **Карточка заказа** редактирует всё, кроме статуса, через PostgREST под RLS; статус — только через RPC-кнопки переходов. Поле «Комментарий для клиента (виден по QR)» отделено от внутреннего комментария мастера.
- **Импорт справочника:** прогресс-бар по пачкам 500 строк, итог «добавлено/дублей», разворачиваемый список построчных ошибок — контракт `import_catalog_batch` с Этапа 5.
