## 2. Отклонения от первоначального плана (все — задокументированы в своих этапах)

1. **PDF не генерируется на сервере** (план Этапа 6) → печатные HTML-формы из снимков (Этап 11): идеальная кириллица, 0 Edge-инвокаций, «Сохранить как PDF» в браузере. `order_documents.storage_path` оставлен под Phase 2 (email-отправка PDF).
2. **Docker-compose для Supabase не писался** → официальный CLI (Этап 14); Docker остался для self-hosted фронтенда.
3. **Создание заказа офлайн не поддержано** (Этап 13): приёмка происходит у стойки с сетью; офлайн-очередь покрывает работу мастера (статусы, диагностика).

## 3. Известные ограничения (осознанные компромиссы)

- Rate limit публичной страницы — в памяти изолята (best effort); токен 128 бит делает перебор бессмысленным.
- Смена роли сотрудника вступает в JWT за ≤1 час или по перелогину; мгновенное отсечение — деактивацией.
- Storage-политики не покрыты локальным тестом (нет схемы storage вне Supabase) — повторяют проверенный паттерн RLS attachments + ручной чек-лист деплоя.
- `xlsx` имеет известную уязвимость без фикса в npm — затрагивает только импорт админом собственных файлов.
- Бэкапы на Free — ручной еженедельный дамп (runbook, шаг 11).
