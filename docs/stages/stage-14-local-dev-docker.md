## 1. Запуск локального стека одной командой

```bash
./scripts/dev.sh   # потом: npm run dev
```

Скрипт делает всё сам:
1. `supabase start` — полный стек Supabase в Docker (Postgres, GoTrue, PostgREST, Storage, Studio) через официальный CLI. Рукописный docker-compose не пишем сознательно: стек из 10+ сервисов с миграциями версий — это то, что CLI поддерживает за нас.
2. `supabase db reset` — все миграции + продовый `supabase/seed.sql` (статусы, переходы, правила, шаблоны полей).
3. `scripts/demo-data.sql` — демо-наполнение.
4. Пишет `.env.local` для Vite из ключей локального стека.

Вход: `admin@demo.local` / `manager@demo.local` / `master@demo.local`, пароль `demo1234`. Studio: http://127.0.0.1:54323.

## 2. Демо-данные: через боевые RPC, а не голые INSERT

`demo-data.sql` создаёт заказы вызовами `create_order` / `quick_add_model` / `change_status` под подставленными JWT-claims трёх ролей — демо проходит **те же триггеры, валидации и уведомления**, что и реальная работа. Получается живая картина:

Демо-пользователи создаются прямо в GoTrue (bcrypt-пароль, identities, `app_metadata.role`); на тестовом шиме без GoTrue ветка автоматически упрощается до голых id — поэтому файл прогоняется и в CI-harness.

## 3. Docker для self-hosted фронтенда

Появился `Dockerfile` (multi-stage: node:22 сборка → nginx:1.27) и `deploy/nginx.conf`:
- SPA-fallback на index.html (включая `/status/<token>`),
- хешированные ассеты Vite — `immutable, max-age=1y`,
- `sw.js` / `manifest.webmanifest` / `index.html` — `no-cache` (обязательное условие корректных обновлений PWA),
- gzip, healthcheck.

Это **опция** для самостоятельного хостинга; базовый путь деплоя (Этап 17) остаётся бесплатным: статика на Vercel/Netlify/Cloudflare Pages + Supabase Cloud.

## 6. Риски и открытые вопросы

- `demo-data.sql` рассчитан на однократный запуск поверх свежей базы (`db reset` в dev.sh это гарантирует); защиты от повторного прогона нет сознательно — простота важнее.
- Вставка в `auth.users` напрямую — приём только для локального GoTrue; в облаке сотрудники создаются через admin-users (Этап 8).
