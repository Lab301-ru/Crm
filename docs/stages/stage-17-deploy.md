## 3. Ключевые решения

- **Seed отдельно от `db push`**: CLI не применяет seed в облако — в runbook это явный шаг с psql/SQL Editor; повторный запуск безопасен (`on conflict do nothing`).
- **Уведомления опциональны при старте**: без SMTP/Telegram система полноценно работает (канал «телефон» — задачи на дашборде); каналы доключаются позже без деплоя.
- **Free-проект не заснёт**: ежеминутный pg_cron — естественный keep-alive (пауза Supabase наступает после 7 дней без запросов).
- **Чек-лист проверяет границы безопасности**, а не только happy-path: публичная страница в инкогнито работает, CRM в инкогнито — нет; мастер не видит финансов.

## 4. Риски и открытые вопросы

- SMTP российских провайдеров (Яндекс 360, Mail.ru) требует «пароль приложения» — в runbook отмечено; If письма не пойдут — канал отключается одним тумблером.
- Привязка Telegram требует действия клиента (нажать Start) — это осознанный opt-in: рассылать без согласия нельзя.
- Бэкап на Free — ручной еженедельный дамп; для бизнеса с >300 заказов/мес рекомендован Pro.
